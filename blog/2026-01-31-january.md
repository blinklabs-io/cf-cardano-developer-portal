Below is a Q1 2026 update highlighting ecosystem tooling and solutions across January, February, and March.

The first quarter of 2026 picked up right where the previous year left off. From major releases and interoperability progress to governance enhancements and developer tooling upgrades, teams across the ecosystem continued to ship at pace. With consistent improvements across infrastructure, analytics, and developer experience, the momentum remains strong. Check out the highlights below to see what’s been delivered.

## Reeve

`cardano-foundation/reeve`

**January**:

* Provenant vLEI publishing and attestation scripts implemented according to CIP-170
* Reeve indexer vLEI verification implemented
* Finalized new reporting module with configurable report templates
* Dev and deployment preparations for upcoming `v1.3.0`

**February**:

* Released versions `v1.3.0` and `v1.3.1`
* Improved reconciliation statistics, filtering, and indexer performance
* Enhanced download endpoints for transactions and reports
* Validation upgrades including duplicate rules, CSV validation, and template protections
* Fixed reconciliation date issues, duplicate key exceptions, and report loop bug
* OpenAPI and Flyway upgrades, GitHub runner migration

**March**:

* Released versions `v1.4.0` and `v1.4.1`
* Extended transaction metadata with parent project code and batchId
* Improved download filters and template configuration options
* Added CustomerCredit transaction type and rejection workflows
* Enhanced reconciliation views and status handling
* Multiple fixes across reconciliation logic and CSV processing

## Cardano IBC Incubator

`cardano-foundation/cardano-ibc-incubator`

**January**:

* Hermes migration, health-check monitoring, and denom-trace PostgreSQL integration
* Added integration tests for Gateway and Hermes
* Consolidated Cosmos-side verification to Mithril, removed legacy light client

**February**:

* Hardened Docker reproducibility across Cardano, Gateway, and Cosmos
* Introduced native message exchange entrypoint
* Modularized Caribic and improved startup consistency
* Fixed packet handling, escrow, collateral, and denom-trace integrity
* Persisted and hydrated Gateway IBC state tree cache
* Hardened ICS-20 flows and improved JSON handling
* Updated Gateway proof contract and improved faucet security
* Extensive integration tests and documentation updates
* Removed legacy light client docs and cleaned unused code

**March**:

* Migrated Gateway data pipeline to Yaci-based data plane
* Upgraded IBC-Go from v8 to v10
* Added bridge discovery manifest and improved transfer planning
* Extended Caribic with Injective testnet and Cheqd support
* Improved Hermes lifecycle and runtime architecture
* Stability fixes for token swaps and Mithril lag handling
* Documentation and integration environment improvements
